# 总体结构：3 个脚本 + 1 个主执行器

* **Script A：Trend（趋势）** → 输出“方向/强度/位置/最合适交易位置”
* **Script B：Space（空间）** → 输出“上下界/空间剩余/止损/盈利/RR”
* **Script C：Trigger（触发）** → 输出“底背离/顶背离/触发优先级/信号新鲜度”
* **Main Exec：Assembler（拼装执行）** → 按规则做：允许开仓/分三段买/两段卖/止损优先

---

# A) Trend 脚本（趋势）

## 输入

* `TF_BIG`（你指定的趋势周期）
* 你内部使用的趋势指标（DSA/均线/结构都行）

## 输出（标准字段）

1. `trend_dir`：`+1 / -1 / 0`
2. `trend_strength`：`0~100` 或 `0~1`（你定）
3. `trend_position`：当前位置（建议归一化到 0~1，0=低位，1=高位；或用 zscore）
4. `best_trade_zone`：这个周期“最合适交易位置”

   * 建议结构化：

     * `best_zone_type`：`"discount" | "neutral" | "premium"`
     * `best_zone_score`：0~1
     * `best_zone_reason`：可选，给 debug

## 产出给主策略的“趋势判定 Gate”

主策略只关心一个布尔：

* `G_BIG = (trend_dir == +1) AND (best_zone_type == "discount")`
  （或你用强度阈值：`trend_strength >= S_th`）

> 这样趋势脚本负责“判断趋势 + 当前处在什么位置 + 是否适合交易”，主策略不再管趋势细节。

---

# B) Space 脚本（空间）

## 输入

* `TF_MID = [m1, m2, ...]`（1~N个中周期）
* 每个中周期的 liquidity zones（按你附件口径：upper_y / lower_y，排除 grabbed）
* ATR（与你 LZ 同口径即可）

## 输出（标准字段）

**1) 上下界（最终合成值 + 源周期）**

* `upper_bound`：最终上界（用 nearest upper y）
* `lower_bound`：最终下界（用 nearest lower y）
* `upper_src_tf`、`lower_src_tf`

合成口径建议固定为“保守”：

* `upper_bound = min(valid uppers)`
* `lower_bound = max(valid lowers)`

**2) 止损与盈利空间**

* `stop_price = lower_bound - SL_BUFFER_ATR * ATR(lower_src_tf)`
* `reward = upper_bound - entry_price`（entry_price 在主策略里传入或用 close）
* `risk = entry_price - stop_price`

**3) RR**

* `rr = reward / risk`
* `rr_pass = (rr >= RR_MIN) AND (reward>0) AND (risk>0)`

**4) 空间剩余（卖出用）**

* `space_left = upper_bound - close`
* `space_left_atr = space_left / ATR(upper_src_tf)`（可选）
* `allow_takeprofit = (space_left_atr <= space_th)`（主策略用这个当止盈前置）

> 这样空间脚本是“纯几何/赔率引擎”：给上下界、止损、RR、剩余空间；不掺背离触发。

---

# C) Trigger 脚本（触发点）

## 输入

* `TF_SMALL = [s1, s2, ...]`
* `SMALL_PRIORITY = [s1, s2, ...]`（左→右优先级）
* 你的小周期背离检测逻辑

## 输出（标准字段）

对每个小周期都输出一份信号，然后再输出一个“最终选择”的触发源：

**1) 每周期信号**

* `bull_div[s]`：底背离 True/False
* `bear_div[s]`：顶背离 True/False
* `div_age[s]`：0~1（越小越新）可选
* `signal_quality[s]`：0~1（可选，用于过滤假信号）

**2) 最终触发选择（按优先级左→右）**

* `buy_trigger_tf`：第一个满足买触发条件的 tf（没有则 None）
* `sell_trigger_tf`：第一个满足卖触发条件的 tf（没有则 None）
* `buy_trigger = (buy_trigger_tf != None)`
* `sell_trigger = (sell_trigger_tf != None)`

买触发条件建议写死一种（可配置阈值）：

* `bull_div == True AND div_age <= AGE_TH`（如果没有 age，就只用 bull_div）

卖触发同理：

* `bear_div == True AND div_age <= AGE_TH`

# Main Exec（拼装执行规则：明确、可落地）

## 1) 开仓许可（一次性总闸）

**允许寻找买点：**

* `trend.G_BIG == True`
* `space.rr_pass == True`
* `trigger.buy_trigger == True`
* `stop_price` 有效（非 None）

满足才进入买入执行。

---

## 2) 三段买入（只由主执行器控制仓位与次数）

设 `stage ∈ {0,1,2,3}`：

* **BUY-1**：`stage==0 AND allow_entry` → 买 `W1`，`stage=1`
* **BUY-2**：`stage==1 AND trend.G_BIG AND space.rr_pass AND trigger.buy_trigger` → 买 `W2`，`stage=2`
* **BUY-3**：`stage==2 AND trend.G_BIG AND space.rr_pass AND trigger.buy_trigger` → 买 `W3`，`stage=3`

关键硬规则：**每一段都要重算一次 space（RR/stop/upper/lower），RR 不过就不加仓。**

> 你后续如果要更严格：BUY-2/3 增加“必须回到折价区附近”之类条件，也只改主执行器，不动三脚本。

---

## 3) 两段卖出（空间前置 + 顶背离触发）

* **SELL-1（允许止盈 + 顶背离触发）**
  条件：`space.allow_takeprofit AND trigger.sell_trigger` → 卖出 50%
* **SELL-2（再次触发或结构走坏）**
  条件（二选一）：

  * 再次满足 `trigger.sell_trigger`
  * 或者 `close` 跌破你定义的执行线（比如小周期下界/短均线/VWAP）→ 卖出剩余

---

## 4) 止损优先级（最高）

任何时刻：

* 若 `low <= space.stop_price`（或 close 跌破）→ **立即全平**
  并且止损优先于任何止盈/加仓信号。

---

# 你最终会得到的“拼装接口”（最重要）

主策略每根 bar 只做三次调用：

1. `trend = trend_script(TF_BIG, data)`
2. `space = space_script(TF_MID, data, entry_price=close)`
3. `trigger = trigger_script(TF_SMALL, SMALL_PRIORITY, data)`
4. 用上面的 Main Exec 规则决策下单
