编码风格规则（统一、可读、易维护）

自动格式化优先：代码必须可被统一 formatter 格式化后保持稳定（不手工对齐、不追求列对齐美观）。

命名统一：函数/变量用 snake_case；类用 PascalCase；常量用 UPPER_SNAKE_CASE；布尔变量用 is_/has_/should_ 前缀。

导入规范：只用显式导入；禁止 import *；导入分组为“标准库→第三方→项目内”，组间空行；删除未使用导入。

函数短小清晰：单函数只做一件事；避免深层嵌套（优先早返回）；同一文件避免超长函数（需要拆 helper）。

显式优于隐式：避免“魔法数字/魔法字符串”，抽成常量或配置；条件判断写清楚，不写晦涩一行流。

一致的返回与异常：同类函数保持一致的返回类型；不要用 except: pass 静默吞错；失败要么抛明确异常，要么返回明确结构。

注释只写关键：注释写“为什么/约束/边界”，不要复述代码；公共函数/类给一句话 docstring 即可。

减少副作用：函数内部不随意修改外部对象；需要修改时显式说明（例如返回新对象或清楚的 in-place 命名）。

日志与打印：库/核心函数禁止 print；如需输出，用统一 logger，且只在最外层脚本记录。

文件组织：一个文件聚焦一个主题；重复工具函数集中到工具模块；避免散落同名/同义 helper。
