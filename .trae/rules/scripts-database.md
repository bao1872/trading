**目标**：任何脚本都“可直接运行”，任何数据库相关改动都“文档可追溯、结构一致”，并支持单脚本快速调试。

1. **脚本运行说明必写**：每个可运行脚本文件顶部必须包含注释/模块 docstring，至少含：
   Purpose（用途一句话）
   Inputs/Outputs（输入输出：文件/参数/表）
   How to Run（运行命令，含 `--help`）
   Examples（≥2条可复制示例）
   Side Effects（是否写库/改表/写文件，必须明确）

2. **可调试入口必备**：每个脚本必须包含 `if __name__ == "__main__":` 入口，用于本脚本函数的本地调试/冒烟运行；入口内应调用 `main()` 或最小示例流程（与上方 Examples 保持一致），避免把核心逻辑写在入口里。

3. **数据库触发条件**：脚本只要读/写数据库，或涉及 schema 变更（建表/删表/增删列/改类型/约束/索引/主键唯一键/写入语义变化），必须同步更新表结构说明。

4. **表结构说明最小字段**：对每张表写清：表名+用途、主键/唯一键、关键索引、字段清单（name/type/null/default/desc）、写入语义（append/replace/upsert及冲突键）、更新时间/版本号。

5. **变更记录必追加**：任何 schema 变更必须在 Schema Changelog 追加一条：日期、涉及脚本、变更摘要、影响范围（哪些脚本依赖/是否需回填）、兼容性与迁移步骤（如不兼容）。

6. **验收标准**：脚本说明块齐全；存在 `__main__` 入口且可运行；涉及 DB 时文档已更新；文档中的表名/字段/键/索引与代码一致。
