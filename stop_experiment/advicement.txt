# 二、你要的目标架构：单引擎，多模式

## 这个文件只暴露两个函数

### 1. `run_day()`

负责单日推进

```python
def run_day(
    date,
    holdings,
    pending_buys,
    pending_sells,
    predictions_df,
    price_pivot,
    trading_days,
    prev_close_map,
    params,
    write_ledgers=False,
    ledger_base_dir=None,
):
    ...
```

它内部只做这几件事：

1. 用统一的 candidate pool builder 生成当日候选池
2. 计算 prev_decision_date
3. 调 `step_day()`
4. 如果 `write_ledgers=True`，就调用 `live_ledger` 写：

   * holdings
   * decisions
   * executions
5. 返回统一结果结构

---

### 2. `run_range()`

负责多日连续运行

```python
def run_range(
    start_date,
    end_date,
    mode,                 # "backtest" | "replay" | "live"
    prediction_source,    # "full_test" | "daily_parquet"
    params,
    initial_holdings=None,
    write_ledgers=True,
    ledger_base_dir=None,
    postprocess=True,
):
    ...
```

它内部只做：

1. 按日期循环
2. 每天读取 predictions
3. 每天调用 `run_day()`
4. 循环结束后调用统一后处理：

   * `build_live_equity_curve`
   * `build_live_trade_report`
   * `12_build_operator_packet`（只在 live 模式）

---

# 三、三种模式怎么定义

## mode = `backtest`

用途：研究基线、历史回测

* `prediction_source = "full_test"`
* 初始持仓 = 空仓
* 输出 ledger 到：

  * `output/backtest_ledger/`
* 最后生成：

  * `baseline_e0_x1_v1_nav.csv`
  * `baseline_e0_x1_v1_trades.csv`
  * equity curve
  * trade report

**注意：backtest 不再自己算净值和交易表，而是从同一套 ledger 重建。**

---

## mode = `replay`

用途：一致性验证

* `prediction_source = "full_test"`
* 初始持仓 = 空仓 / 指定起始状态
* 输出 ledger 到：

  * `output/replay_ledger/`
* 只用于跟 backtest 对账

---

## mode = `live`

用途：每日盘后生产

* `prediction_source = "daily_parquet"`
* 初始持仓 = `load_holdings()`
* 输出 ledger 到：

  * `output/live/...`
* 自动生成：

  * equity curve
  * trade report
  * action plan
  * operator packet

---
