你真正需要验证的不是：

```text
这个策略能不能独立跑出漂亮 NAV
```

而是：

```text
这个模型筛出来的前 20 / 前 30，是否比全市场/全候选池更值得你花时间看？
```

所以重点从“账户回测”改成“过滤器质量评估”。

---

## 最小够用版：只做 4 个检查

### 1. TopK 命中率

每天取模型：

```text
Top10 / Top20 / Top30
```

看未来：

```text
1日、3日、5日、10日
```

表现是否优于候选池平均水平。

你只需要看表：

| 组别    | 次日上涨率 | 3日上涨率 | 5日上涨率 | 5日平均收益 | 5日最大回撤 |
| ----- | ----: | ----: | ----: | -----: | -----: |
| Top10 |       |       |       |        |        |
| Top20 |       |       |       |        |        |
| Top30 |       |       |       |        |        |
| 全候选池  |       |       |       |        |        |

如果 Top20/Top30 稳定优于全候选池，就够了。

---

### 2. 分数分桶

把每天候选股按模型分数分成 5 桶：

```text
最高分20%
次高分20%
中间20%
次低20%
最低20%
```

看未来收益。

你只需要判断：

```text
最高分这一桶是不是明显优于最低分这一桶？
```

不要求严格单调，但至少应该看到：

```text
高分组 > 低分组
```

如果连这个都没有，那模型过滤价值不大。

---

### 3. TopK 覆盖优质股能力

你主观选股时最关心的是：

```text
好票有没有被模型筛进来？
```

所以可以定义：

```text
未来5日涨幅 > 5%
未来10日涨幅 > 8%
未来20日 MFE > 10%
```

然后看这些“事后好票”有多少出现在 Top30 / Top50。

例如：

| 标准         | 全候选好票数 | Top30覆盖数 | 覆盖率 |
| ---------- | -----: | -------: | --: |
| 5日涨幅>5%    |    100 |       28 | 28% |
| 10日涨幅>8%   |     80 |       25 | 31% |
| 20日MFE>10% |    120 |       40 | 33% |

如果 Top30 只占全候选池很小比例，却能覆盖较多好票，说明过滤器有效。

---

### 4. 排除烂票能力

模型不只是找好票，也要帮你少看烂票。

看 Top30 里：

```text
未来5日最大回撤 < -5%
未来10日最大回撤 < -8%
```

的比例是否低于全候选池。

如果 Top30 的回撤风险明显低，说明它能帮你省时间。

---

# 你不需要重点看 NAV

如果你最后是主观挑选，NAV 反而容易误导。

因为 NAV 假设：

```text
每天机械买入
固定仓位
固定退出
不主观筛选
不考虑题材/消息/盘口
```

这和你的实际用法不一致。

你真正要的是：

```text
模型能不能把 500 个候选缩到 30 个更值得研究的候选
```

所以核心指标应该是：

```text
Top30质量
Top30覆盖率
Top30风险
高分组 vs 低分组
```

---

## 我建议的最终评估方式

每天输出一个“过滤器日报”：

| 字段               | 含义     |
| ---------------- | ------ |
| 日期               |        |
| 候选池数量            | 比如 400 |
| Top10平均分         |        |
| Top30平均分         |        |
| Top30未来5日平均收益    | 回测用    |
| Top30未来5日胜率      | 回测用    |
| Top30未来5日最大回撤中位数 | 回测用    |
| 全候选池未来5日平均收益     | 对照     |
| Top30相对全池超额收益    | 核心     |
| Top30好票覆盖率       | 核心     |
| Top30坏票占比        | 核心     |

实盘当天只输出：

```text
Top30候选
每只股票的模型理由
风险提示
你再主观筛
```
