## 实验记录

### 1. 候选观察窗口对比实验

| 项目 | 内容 |
|---|---|
| 脚本 | `obs_window_comparison.py` |
| 目的 | 验证 candidate_obs_days 窗口长度对回测表现的影响 |
| 方案 | C3=[1,2,3], C5=[1..5], C10=[1..10], C20=[1..20] |
| 结论 | **C3 最优**: NAV 2.74, 胜率 82.9% > C20 NAV 2.53, 胜率 75.8% |
| 状态 | ✅ 已验证，参数已固化到 `stop_config.py` |

### 2. 组合决策回测实验（准入过滤）

| 项目 | 内容 |
|---|---|
| 脚本 | `portfolio_policy_backtest.py` |
| 目的 | 验证 Tier 分层是否适合做准入门槛 |
| 方案 | 保守型(只买 A+B, max=6) / 平衡型(A+B+C, max=10) / 进攻型(A+B+C, max=12) |
| 结论 | **全部跑输基线**: 最佳进攻型 NAV -8.06%。Tier A 机会太少(~5%)，过滤掉大量盈利机会 |
| 状态 | ❌ 已证伪，Tier 不适合做准入过滤 |
| 踩坑 | 基线 NAV 不一致 → 修复 weight 复衡逻辑（卖出后不复衡） |

### 3. Tier 差异化仓位映射实验

| 项目 | 内容 |
|---|---|
| 脚本 | `tier_weight_mapping.py` |
| 目的 | 验证 Tier 分层是否适合做仓位映射（"买多少"而非"能不能买"） |
| 方案 | W1(A:1.3,B:1.0,C:0.7) / W2(A:1.5,B:1.0,C:0.5) / W3(A:2.0,B:1.0,C:0.3) |
| 结论 | **W3 最优**: NAV +3.89% vs 基线，W1 NAV +1.34%。Tier 适合做仓位控制 |
| 状态 | ✅ 当前最优方案，已接入日报 `08_daily_inference_report.py` |
| 应用 | W1 = 低风险基准版（模拟盘主用），W3 = 高收益实验版（对照组） |

## 实验方法论

1. **基线对齐优先**: 任何策略实验必须先验证 baseline NAV 与原始回测一致
2. **单一变量原则**: 每次只改一个维度（窗口/过滤/权重），保持其他不变
3. **SSOT 不可触碰**: 卖出决策（max_hold/stop_loss/model_risk）和候选池构建不可改
4. **结论驱动**: 实验脚本头部注释必须包含结论，不能只写"做实验"
