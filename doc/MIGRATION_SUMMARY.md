# 统一调度器迁移总结

## 概述

已将原分散在 `app/quote_snapshot.py` 等文件中的定时调度逻辑提取到独立的 `scheduler.py` 文件中，实现集中化管理。

## 文件结构

```
交易/
├── scheduler.py              # 统一调度器（新建）
├── .env                      # 环境变量配置（新建）
├── .env.example              # 配置模板（新建）
├── scheduler.log             # 调度器日志（自动生成）
├── docs/
│   └── scheduler.md          # 使用手册（新建）
├── app/
│   └── quote_snapshot.py     # 背离扫描业务逻辑（保留）
└── ...
```

## 主要改进

### 1. 职责分离 ✅
- **调度器** (`scheduler.py`): 只负责任务调度和启停管理
- **业务逻辑** (`app/quote_snapshot.py`): 专注背离扫描等具体业务
- 符合 `dispatch.md` 规则要求

### 2. 集中管理 ✅
所有任务在一个地方配置：
- 背离扫描任务
- 临时文件清理任务
- 日报生成任务（预留）

### 3. 配置外部化 ✅
通过 `.env` 文件管理配置：
```bash
ENABLE_DIVERGENCE_SCAN=true      # 背离扫描开关
ENABLE_CLEANUP=true             # 文件清理开关
DIVERGENCE_SCAN_INTERVAL=5      # 扫描间隔
```

### 4. 日志记录 ✅
统一日志格式：
```
2026-03-05 13:31:43,067 - scheduler - INFO - 统一任务调度器启动
2026-03-05 13:31:43,071 - scheduler - INFO - 注册任务：背离扫描
2026-03-05 13:31:43,072 - scheduler - INFO - 启动调度器...
```

### 5. 优雅退出 ✅
捕获 SIGTERM/SIGINT 信号，完成当前任务后再退出

### 6. 防止并发 ✅
设置 `max_instances=1`，防止同一任务并发执行

## 任务配置

### 背离扫描任务
```python
trigger='cron',
minute='*/5',              # 每 5 分钟
hour='9-11,13-15',        # 交易时段
day_of_week='mon-fri'     # 工作日
```

### 临时文件清理任务
```python
trigger='cron',
hour=17,                   # 每天 17:00
minute=0,
day_of_week='mon-fri'
```

## 使用方式

### 启动调度器
```bash
# 后台运行
nohup python scheduler.py > scheduler.log 2>&1 &

# 查看状态
ps aux | grep scheduler.py

# 查看日志
tail -f scheduler.log
```

### 停止调度器
```bash
# 优雅停止
kill <PID>

# 强制停止
kill -9 <PID>
```

### 配置任务
编辑 `.env` 文件：
```bash
# 开启背离扫描
ENABLE_DIVERGENCE_SCAN=true

# 关闭日报生成
ENABLE_DAILY_REPORT=false
```
