# PRD - 量化交易策略后端系统

## 一、项目概述

### 1.1 项目背景
本项目是一个量化交易策略后端系统，旨在为A股市场提供高效的数据获取、特征计算和策略分析能力。系统通过通达信行情接口获取K线数据，计算多种技术指标特征，并支持可视化分析。

### 1.2 项目目标
- 建立稳定可靠的K线数据获取与存储体系
- 实现多种技术指标特征的自动化扫描与入库
- 提供可扩展的技术指标可视化分析工具
- 为后续策略开发和交易执行提供数据支撑

### 1.3 目标用户
- 量化策略研究员
- 个人量化交易者

---

## 二、架构设计理念

### 2.1 核心理念

```
cores/  →  实验场（保持原样，自由探索，允许快速迭代与验证新算法、新功能）
     ↓
src/          →  工程化（引用 cores 中的核心函数，提供统一接口，确保代码质量与可维护性）
```

### 2.2 目录定位

| 目录 | 定位 | 特点 |
|------|------|------|
| `cores/` | 实验场 | 保持原样，自由探索，允许快速迭代与验证新算法、新功能 |
| `src/` | 工程化 | 引用 cores 中的核心函数，提供统一接口，确保代码质量与可维护性 |

### 2.3 引用规范

- **cores**：保持原样，继续作为算法实验场
- **src/scanner**：模块需明确引用 cores 中的核心函数（如 `compute_indicators`、`calculate_divs` 等），并对输入输出参数进行标准化处理

### 2.4 不建议改动的地方

1. **cores 内部结构**：保持其原有的组织方式，确保实验的自由度与灵活性
2. **cores 中的 pytdx 调用**：允许实验脚本直接使用 pytdx 获取数据，满足快速验证需求
3. **cores 的可视化代码**：保留所有可视化相关代码，支持算法效果的直观验证与调整

### 3.3 股票池文件说明

项目使用两个不同的股票池Excel文件，用途各异：

| 文件 | 用途 | 使用场景 |
|------|------|----------|
| `stock.xlsx` | **自选股列表** | 回测、实时监控自选股的技术特征状态 |
| `stock_concepts_cache.xlsx` | **全市场品种属性** | 选股筛选，包含概念、人气、市值等属性 |

**设计原因**：
- **选股场景**：需要全市场品种的属性数据（概念板块、人气排名、市值等），用于筛选符合条件的标的
- **监控场景**：只需关注已选定的自选股，减少计算量，聚焦核心持仓或关注标的

## 四、功能模块

### 4.1 数据库对接模块 (db.py)

标准化的数据库对接模块，所有数据库操作统一引用此模块。

| 功能 | 描述 |
|------|------|
| 连接管理 | 统一的数据库连接池管理 |
| 会话管理 | 上下文管理器自动处理会话生命周期 |
| 批量写入 | 高效的批量数据写入（支持 upsert） |
| 批量查询 | 支持条件查询、分页查询 |
| 表清空 | 支持 TRUNCATE 操作 |
| 事务支持 | 支持事务操作，自动回滚 |

**核心接口**：

```python
from src.db import get_session, bulk_upsert, truncate_table, query_df

# 会话管理（上下文管理器）
with get_session() as session:
    # 执行数据库操作
    pass

# 批量写入（upsert 模式）
bulk_upsert(
    session=session,
    model=StockDivFeatures,
    df=features_df,
    unique_keys=["ts_code", "freq", "bar_time"]
)

# 清空表
truncate_table(session, StockDivFeatures)

# 查询返回 DataFrame
df = query_df(
    session=session,
    model=StockDivFeatures,
    filters={"ts_code": "600547.SH", "freq": "d"},
    order_by="bar_time DESC",
    limit=100
)
```

**设计原则**：
- 单一入口：所有数据库操作通过此模块
- 自动重连：连接断开时自动重连
- 错误处理：统一的异常处理和日志记录
- 性能优化：使用连接池和批量操作

### 4.4 K线数据管理

> **重要约束**：
> - 行情数据的保存**只能**使用 `k_data_saver.py`
> - 行情数据的获取**统一调用** `k_data_loader.py`
> - **禁止**其他任何形式单独获取行情数据（如直接调用pytdx、其他数据源等）
> - 所有模块需要行情数据时，必须通过 `k_data_loader.load_k_data()` 接口获取

#### 4.4.1 数据获取 (k_data_saver.py)

| 功能 | 描述 |
|------|------|
| 全量回补 | 通过 `--full` 参数获取指定天数的历史数据（默认3年） |
| 增量更新 | 通过 `--incremental` 参数只更新最后一个索引日期之后的数据 |
| 异步保存 | 后台线程处理保存任务，不影响pytdx读取效率 |
| 断线重连 | 自动检测连接状态，断线时切换到其他服务器 |
| CSV索引 | 维护 `index.csv` 快速定位数据文件 |

**使用方式**：
```bash
# 全量回补
python -m src.k_data_saver --full

# 增量更新
python -m src.k_data_saver --incremental

# 指定参数
python -m src.k_data_saver --full --days 365 --limit 100 --filter "黄金"
```

#### 4.4.2 数据加载 (k_data_loader.py)

| 功能 | 描述 |
|------|------|
| 多周期支持 | 从5分钟原始数据重采样到 15m/30m/60m/d/w 周期 |
| 索引加速 | 通过CSV索引快速定位数据文件 |
| 数据范围查询 | 查询指定股票的数据起止时间 |

**使用方式**：
```bash
# 加载指定股票数据
python -m src.k_data_loader --code 600489.SH --start 2025-01-01 --end 2026-02-15 --freq 60m

# 列出所有可用股票
python -m src.k_data_loader --list
```

### 4.5 特征扫描

#### 4.5.1 背离特征扫描 (divergence_scanner.py)

| 功能 | 描述 |
|------|------|
| 多指标支持 | MACD、HIST、OBV 三种指标的背离检测 |
| 四种背离类型 | 底背离(0)、顶背离(1)、隐藏底背离(2)、隐藏顶背离(3) |
| 多周期支持 | 15m、60m、d、w 等周期 |
| 向量化计算 | 使用向量化版本的 `pivots_confirmed` 提升性能 |

**数据库表结构**：
| 字段 | 类型 | 描述 |
|------|------|------|
| ts_code | VARCHAR(20) | 股票代码 |
| freq | VARCHAR(10) | 周期 |
| bar_time | TIMESTAMP | K线时间 |
| macd_has_div | INTEGER | MACD是否背离 |
| macd_div_type | INTEGER | MACD背离类型 |
| macd_div_len | INTEGER | MACD背离长度 |

**使用方式**：
```bash
python -m src.divergence_scanner --freqs "15m,60m,d" --filter "中金黄金"
```

#### 4.5.2 AMP特征扫描 (amp_scanner.py)

| 功能 | 描述 |
|------|------|
| 自适应周期 | 从候选周期中选择Pearson相关系数最高的 |
| 通道特征 | 计算通道位置、斜率、强度等 |
| 成交量分析 | 计算成交量活跃度位置 |

**关键特征字段**：
| 字段 | 描述 |
|------|------|
| close_pos_0_1 | 收盘价在通道内的位置（0~1） |
| activity_pos_0_1 | 成交量活跃度位置 |
| strength_pr | Pearson相关系数强度 |
| upper/lower_ret_per_bar | 上下轨每bar收益率 |

**使用方式**：
```bash
python -m src.amp_scanner --freqs "15m,60m,d,w" --filter "中金黄金"
```
