## 常用脚本运行

### 在项目根目录运行脚本

由于项目使用相对导入，需要在项目根目录运行脚本：

```bash
cd /path/to/交易
PYTHONPATH=/path/to/交易:$PYTHONPATH python <script_path> [options]
```

### 1. AMP 扫描

```bash
# 扫描单个股票
PYTHONPATH=/path/to/交易:$PYTHONPATH python app/amp_scanner.py --symbol 600547 --years 3

# 扫描股票池
PYTHONPATH=/path/to/交易:$PYTHONPATH python app/amp_scanner.py --pool all --years 1
```

### 2. 背离检测

```bash
# 检测背离并生成可视化
PYTHONPATH=/path/to/交易:$PYTHONPATH python cores/divergence_many_plotly.py \
  --symbol 600489 \
  --freq 60m \
  --start "2025-11-01 09:30" \
  --end "2026-01-20 15:00" \
  --prd 5 \
  --searchdiv Regular \
  --out output/div.html
```

### 3. 成交量 ZScore

```bash
# 生成成交量 ZScore 可视化
PYTHONPATH=/path/to/交易:$PYTHONPATH python cores/volume_zscore_plotly.py \
  --symbol 600547 \
  --years 3 \
  --win 20 \
  --out output/volz.html
```

### 4. 布林带特征

```bash
# 生成布林带特征可视化
PYTHONPATH=/path/to/交易:$PYTHONPATH python cores/bollinger_features_plotly.py \
  --symbol 600547 \
  --years 3 \
  --bb_win 20 \
  --bb_k 2 \
  --z_win 255 \
  --out output/bb_feat.html
```

## 数据获取

### 统一数据源模块

所有数据获取必须通过 `datasource.pytdx_client` 模块：

```python
from datasource.pytdx_client import connect_pytdx, PERIOD_MAP

def fetch_daily_data(symbol: str, start: str, end: str) -> pd.DataFrame:
    api = connect_pytdx()
    try:
        cat = PERIOD_MAP['d']  # 使用统一映射
        mkt = 1 if symbol.startswith("6") else 0
        # ... 获取数据逻辑
    finally:
        api.disconnect()
```

### 支持的时间周期

- `'1m'`: 1分钟
- `'5m'`: 5分钟  
- `'15m'`: 15分钟
- `'30m'`: 30分钟
- `'60m'`: 60分钟
- `'d'`: 日线
- `'w'`: 周线
- `'m'`: 月线

## 数据库操作

### 数据库模型

主要数据表：
- `stock_list`: 股票列表
- `amp_features`: AMP 特征
- `divergence_signals`: 背离信号
- `volume_zscore`: 成交量 ZScore
- `bollinger_features`: 布林带特征

### 数据库操作示例

```python
from app.db import get_db_connection

# 查询数据
conn = get_db_connection()
df = pd.read_sql_query("SELECT * FROM amp_features WHERE symbol = ?", conn, params=['600547'])

# 插入数据
cursor = conn.cursor()
cursor.execute("INSERT INTO amp_features (symbol, date, feature) VALUES (?, ?, ?)", 
              ('600547', '2024-01-01', 1.23))
conn.commit()
```

## 开发规范

### 新建脚本要求

1. **位置**: 放在 `app/` 目录
2. **文档**: 包含模块 docstring
   ```python
   """
   Purpose: 简要说明脚本用途
   Inputs: 输入参数说明
   Outputs: 输出结果说明
   How to Run: 运行命令示例
   Side Effects: 副作用说明
   """
   ```

3. **入口**: 包含 `if __name__ == "__main__":` 入口

4. **数据获取**: 使用 `datasource.pytdx_client` 模块

### 代码质量要求

- 遵循 PEP 8 编码规范
- 使用 snake_case 命名
- 函数短小清晰，避免深层嵌套
- 异常处理要明确，不使用 `except: pass`
- 避免魔法数字，提取为常量

### 测试要求

- 每个模块末尾包含 `if __name__ == "__main__":` 自测入口
- 测试代码要短小、测核心、无副作用
- 运行命令示例：`python <module.py> --help`

## 临时文件管理

### 清理临时文件

所有临时输出文件放在 `output/` 目录，定期清理：

```bash
# 清理旧的 HTML/PNG 文件
find output/ -name "*.html" -mtime +7 -delete
find output/ -name "*.png" -mtime +7 -delete

# 清理日志文件
find output/ -name "*.log" -mtime +30 -delete
```

### Python 缓存清理

```bash
find . -name "__pycache__" -type d -exec rm -rf {} +
find . -name "*.pyc" -delete
find . -name "*.pyo" -delete
```

## 故障排除

### 常见问题

1. **ImportError**: 确保在项目根目录运行脚本，并设置 PYTHONPATH
2. **pytdx 连接失败**: 检查网络连接，pytdx_client 会自动尝试多个服务器
3. **内存不足**: 对于大数据集，考虑分批处理
4. **权限问题**: 确保有写入 output/ 目录的权限

### 调试技巧

1. **启用详细日志**:
   ```bash
   python script.py --verbose
   ```

2. **单步调试**:
   在脚本中添加 print 语句或使用调试器

3. **检查数据完整性**:
   ```python
   print(f"DataFrame shape: {df.shape}")
   print(f"Missing values: {df.isnull().sum().sum()}")
   ```

## 性能优化

### 数据处理优化

- 优先使用 pandas/numpy 向量化操作，避免 for 循环
- 对于必须的循环，使用 tqdm 显示进度条
- 合理使用数据类型，避免不必要的内存占用

### 离线批处理

- 对于大批量处理，使用 tqdm 显示进度
- 分批处理大数据集，避免内存溢出
- 考虑使用多进程并行处理

### 在线服务优化

- 尽量消除 for 循环，使用向量化操作
- 缓存频繁访问的数据
- 使用连接池管理数据库连接
